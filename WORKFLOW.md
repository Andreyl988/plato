# Правила работы с репозиторием plato.estate

## Принцип ветвления

**Главное правило:** ветка `main` — это всегда рабочая, боевая версия сайта. Она не трогается напрямую. Все изменения вносятся только через отдельные ветки.

## Структура веток

| Ветка | Назначение |
|---|---|
| `main` | Боевая версия сайта (v1.0). Не редактировать напрямую. |
| `v2.0` | Текущая разрабатываемая версия с улучшениями. |
| `cambodia-landing` | Отдельный лендинг по Камбодже (маркетплейс). Создаётся отдельно. |

## Процесс внесения изменений

1. **Никогда не коммитить напрямую в `main`.**
2. Все новые фичи и правки — в ветку `v2.0` (или новую ветку от неё).
3. После тестирования и согласования — делать Pull Request из `v2.0` → `main`.
4. Перед мержем обязательно проверить сайт локально.

## Откат к предыдущей версии

Если что-то пошло не так после деплоя `v2.0`:
```bash
# Переключиться обратно на main (v1.0) в GitHub Pages
git checkout main
# Или откатить main к конкретному коммиту:
git revert HEAD
git push origin main
```
